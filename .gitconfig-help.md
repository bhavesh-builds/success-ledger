## Authentication Options

### Option 1: Personal Access Token (Recommended for HTTPS)

1. Go to GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. Click "Generate new token (classic)"
3. Give it a name (e.g., "Success Ledger")
4. Select scopes: `repo` (full control of private repositories)
5. Click "Generate token"
6. Copy the token (you'll only see it once!)

**First time you push:**
- When prompted for password, use the token instead of your GitHub password
- macOS Keychain will save it for future use

### Option 2: GitHub CLI (Easiest)

Install GitHub CLI:
```bash
brew install gh
```

Authenticate:
```bash
gh auth login
```

This will handle authentication automatically.

### Option 3: SSH Keys (Most Secure)

1. Generate SSH key:
```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```

2. Add to SSH agent:
```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

3. Copy public key:
```bash
cat ~/.ssh/id_ed25519.pub
```

4. Add to GitHub:
   - Go to GitHub → Settings → SSH and GPG keys
   - Click "New SSH key"
   - Paste your public key

5. Change remote to SSH:
```bash
git remote set-url origin git@github.com:bhavesh-builds/success-ledger.git
```



```bash

```

### Push to GitHub
```bash

```

### Pull Latest Changes
```bash
git pull
```

## Troubleshooting

### If you get authentication errors:
1. Check your credentials in macOS Keychain Access app
2. Try using a Personal Access Token
3. Or switch to SSH keys (Option 3 above)

### If you need to update your email:
```bash
git config --global user.email "your_email@example.com"
```

### View all git config:
```bash
git config --global --list
```
